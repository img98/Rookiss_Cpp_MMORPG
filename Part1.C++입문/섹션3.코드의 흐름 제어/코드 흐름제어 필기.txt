[ 열거형 ] enum : 흐름제어와는 무관하지만, 알아두면 좋을 방법
enum은 숫자를 지정안해주면 첫값은 0, 그리고 다음값은 1,2,3,...으로 늘어난다.
const A =1;의 경우, 어셈블리에서 앞으로 나올 A를 1로 취급하는 경우도 있지만, 변수들을 메모리에 올리는 경우도 있다. 이러면 메모리 낭비가 생길수있는데,
enum의 경우 완벽하게 해당 변수를 컴파일러가 해당 숫자로 취급하기에, 그래서 열거형이 아무리 커지더라도 메모리 낭비가 적다.

#이 붙은것은 전처리 지시문 (컴파일이 일어나기 전에 일어난다.)
#define 은 해당 코드를 원하는 숫자,기능 으로 사용할 수 있다. //사실 상수를 만들기 위함보다는, 코드를 바꿔주기 위해 사용함
